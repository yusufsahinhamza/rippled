Detailed Description

template<class TIn, class TOut>
class ripple::FlowOfferStream< TIn, TOut >

Presents and consumes the offers in an order book.

The ‘view_’ ApplyViewaccumulates changes to the ledger. ThecancelView_` is used to determine if an offer is found unfunded or became unfunded. The permToRemove collection identifies offers that should be removed even if the strand associated with this OfferStream is not applied.

Certain invalid offers are added to the permToRemove collection:

  • Offers with missing ledger entries
  • Offers that expired
  • Offers found unfunded: An offer is found unfunded when the corresponding balance is zero and the caller has not modified the balance. This is accomplished by also looking up the balance in the cancel view.
